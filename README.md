# README

### 1. 프로젝트 개요 및 목적
* **개발 목적:** 다중 오디오 소스 구동 환경에서 발생할 수 있는 소리 겹침 현상을 제어하고 관리하기 위함.
* **사용 예시:** 화상 회의(Zoom, Teams 등) 진행 중 윈도우 볼륨 믹서를 직접 조작하는 번거로움 없이, 마우스 포커스 전환만으로 유튜브 등의 배경 오디오 소리를 즉각 감쇠시킴으로써 소리 겹침 없는 효율적인 전환 및 청취가 가능함.

---

### 2. 제어 메커니즘
* **프로세스 ID(PID) 기반 제어:** 윈도우 운영체제에서 사용자가 현재 선택하여 최상단에 위치한 활성 창의 프로세스 ID(PID)를 실시간 추적함.
* **볼륨 비중 자동 전환:** 포커스된 PID에 해당하는 어플리케이션의 오디오 세션 볼륨 비중은 지정된 고볼륨으로 유지하고, 그 외 배경 프로세스의 볼륨 비중은 사전에 설정한 저볼륨 임계값으로 즉각 감쇠 처리함.

---

### 3. 사용 방법 (Usage)

#### 가. 볼륨 임계값 설정
1. 프로그램 중앙의 **'집중 시 볼륨 (%)'** 및 **'비집중 시 볼륨 (%)'** 영역에서 원하는 볼륨 비율을 지정함.
2. 마우스 드래그(슬라이더) 또는 우측 숫자 박스에 키보드로 직접 수치를 입력하여 설정할 수 있음.
3. 수치 입력 후 **[설정 적용]** 버튼을 클릭하여 소리 제어 엔진에 해당 볼륨 비중을 반영함.

#### 나. 엔진 구동 및 정지
* **엔진 시작:** 하단의 **[Start]** 버튼을 클릭 시 백그라운드 훅 스레드가 가동되며 상단 상태바가 `상태: 작동 중 🟢`으로 변경되고 자동 볼륨 제어가 시작됨.
* **엔진 정지:** 하단의 **[Stop]** 버튼을 클릭 시 이벤트 훅이 즉시 해제되며 상단 상태바가 `상태: 대기 중 ⚪`으로 변경되고, 제어 중이던 모든 프로세스의 소리가 프로그램 구동 전의 원래 볼륨으로 원상 복구됨.

#### 다. 프로그램 종료
* [Stop] 버튼을 누르지 않고 프로그램 창의 **[X]** 버튼을 눌러 종료하더라도, 내부 안전 종료 루틴(`OnClosing`)이 트리거되어 제어 중이던 시스템 오디오 상태를 자동으로 안전하게 복구함.
